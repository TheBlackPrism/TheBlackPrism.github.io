Solving Sudoku with CSP and creating a dragnet with Datalog

By Jennifer Schürch & Yves Lütjens

Sudoku

Writing a programm to solve any possible Sudoku riddle can rather easely be done using the Constraint Satisfaction Problem. By adding only a few constraints one can solve a Sudoku in no time. We created two different approaches and let them run against each other.

Domain Adaption

In this approach we focused on the adaption of the domain, which means that we only added the all different constraints that each number can only occure once. To insert the existing variables we restricted the domain of the field to the given number.

Constraint Addition

In contrary to the solution that has been discussed before, in this approach we add another constraint instead of reducing the domain for existing nubmers. this constraint consits of a lambda function which is added to each predefined variable and limits it's assignment to the given number.

Comparison

Average Time over 10'000 iterations
Approach Average Time per Game [s]
Domain Adaption 0.01889
Constraint Addition 0.01762

The table above indicates, that the approach with the Domain Adaption is a bit slower than the Constraint Addition. Even if the difference is in the range of 0.001 seconds it is clear that even such a small change can influence the performance significantly.
